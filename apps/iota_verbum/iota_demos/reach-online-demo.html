How the Reach Online pipeline works

Behind the scenes, a simple API keeps all the pieces aligned: YouTube, Dropbox, sermons archive and discipleship content.

  1. 1
    Media triggers the pipeline once.
    On Sunday afternoon or Monday morning, media enters the sermon title, passage, preacher and start/end times. The API pulls the latest live, trims the sermon and uploads it with the right title and series.
  2. 2
    Files are tidied and linked.
    The live copy goes to /KidsTalk on Dropbox, the sermon audio is moved to /Sermons, and a simple JSON entry is added to the Reach sermon archive.
  3. 3
    Discipleship content is generated.
    From the same metadata, the API returns a website block, social caption, email snippet, community-group guide and a kids/family follow-up card — all anchored in the actual passage preached.
